Expressions used to find and filter data

For many feature's views, you can provide expressions to:

  • Filter relevant data in a view

  • Find specific data in a view that satisfies a condition

An expression consists of at least one Operator and one or more Values. Expressions can be used in the find and filter operations to more precisely specify desired search or filter criteria. See Regular Expression Syntax for how to specify a find/filter criteria with a regular expression.

Values

Note: Spaces (blanks) in table column names must be substituted with underscore characters when used in regular expressions.

Value Type

Description

Example

Data Fields

Field names from data records that display in table views

Previous values are obtained by using the Data Field name and the suffix ‘$n’, where n = 0, current value
n = 1, previous record value
n = 2, second previous record value

Previous valid values are obtained by using the Data Field name and the suffix '$valid'

Bit Field

Syntax I[n…m], where I is the identifier, n is the starting bit and m is the ending bit.

X[0..7] represents the first 8 bits of “X”

Array

Syntax I[n], where I is the array and n is the element in the array. This value typically used when working with Data Records which contain Block Type fields, that are arrays.

A[1] represents the second element of array “A”.
