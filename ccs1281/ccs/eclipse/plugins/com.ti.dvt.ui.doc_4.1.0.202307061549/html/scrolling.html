Cursor and Scroll Lock

Data views scroll to the very end when new data is received, if user spots a point of interest in the view while data is updating, clicking on the graph or table, will stop the automatic scroll to the end and stays where user has clicked / cursor position, while data continues to get appended to the end.

To continue scrolling to the end, so that cursor position gets updated whenever new data is received, toggle off the scroll lock button on the toolbar.

Please note, if grouping on data views is enabled (the group button on the toolbar: is toggled on), data views will scroll synchronously however the scroll lock currently works independently of the group and will not lock the scrolling of grouped views.

Use the Freeze button on the toolbar, to freeze the data updates and automatic refresh of the data views completely.
