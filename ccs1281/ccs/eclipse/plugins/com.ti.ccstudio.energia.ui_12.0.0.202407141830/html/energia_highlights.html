Energia

Energia support in CCS allows you to import Energia sketches (or .ino files) into CCS. Once imported, you will have access to advanced debugging capabilities through JTAG for your project.

Introduction

You can import Energia sketches into CCS. "Import Energia Sketch" menu will create a CCS project that contains your sketch file. .ino file will be renamed to .c file and included in a new project. During the same import step CCS will also create a library project that contains Energia framework. Both projects will be shown in CCS Project Explorer. You will need to have Energia installed on your system to be able to import sketches. CCS will leverage your Energia installation location to obtain Energia framework source files and MSP-GCC compiler that will be used to build your projects.

How to get started:

  • Start CCS and go to File->Import..., then select Energia | Energia Sketch. The first time, the wizard will ask you to enter the path to where Energia is installed.
    • The directory specified should contain subdirectories such as examples/, hardware/, tools/.
  • Once you specify Energia location, a checkbox to use a simplified Energia perspective will be enabled. It is recommended that you use this option.
    • If you would like to experience full capabilities of Code Composer Studio, then please disable this checkbox. You may also change this preference at any time by opening Window->Preferences, then navigating to the Energia category.
  • Select Next > and browse to your sketch file.
    • You may also import one of the existing Energia example sketches. You will need to select the device that you are working with. Device selection is stored in a project, thus you may have different projects configured for different devices.
  • If you have your device connected, you can how hit "Debug" button in main toolbar to build and start debugger.
    • You will be able to step through your code and examine variables using variables or expression view. Open from "View" menu.
  • Click on red square to stop debugging, e.g. when you would like to make code changes, rebuild and start debugging again.
