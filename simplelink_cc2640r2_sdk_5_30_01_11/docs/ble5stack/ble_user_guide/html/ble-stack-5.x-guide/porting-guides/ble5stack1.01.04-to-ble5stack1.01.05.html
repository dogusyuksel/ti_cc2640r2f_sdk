BLE5-Stack 1.01.04 (SDK 2.40) to BLE5-Stack 1.01.05 (SDK 3.10)¶

This section will describe a way to migrate a project from BLE5-Stack 1.01.04 to a BLE5-Stack 1.01.05 project.

For this migration guide, simple_peripheral from BLE5-Stack 1.01.04 will be ported over to BLE5-Stack 1.01.05. Because the directory structure is nearly identical between the two releases, the recommended approach is to start with a BLE5-Stack 1.01.04 project that contains the same base functionality as the porting target project and merge in any custom functionality.

  1. Choose a BLE5-Stack 1.01.05 example project that contains your target project’s base functionality.

    For reference, see available sample projects that start with simple_

    In this example, we’re going to use simple_peripheral as the starting BLE5-Stack 1.01.05 sample project.

  2. Transfer all modified application files from BLE5-Stack 1.01.04 into the BLE5-Stack 1.01.05 example project.

    In this example, the following files from BLE5-Stack 1.01.04 were moved into simple_peripheral BLE5-Stack 1.01.05 example:

    • simple_peripheral.c

    • simple_peripheral.h

  3. Modify main.c in the BLE5-Stack 1.01.05 example if additional tasks were added in the BLE5-Stack 1.01.04 project.

  4. If necessary, update the project to use the newer TI-RTOS drivers that are supplied with the SimpleLink CC2640R2 SDK.

  5. Refer to the Core SDK release notes for additional information and the TI-RTOS examples included with SimpleLink CC2640R2 SDK.

    For additional information on how BLE5-Stack 1.01.05 uses TI-RTOS see TI-RTOS (RTOS Kernel) Overview

    For any utilized TI Drivers, review TI-RTOS Kernel (SYS/BIOS) User’s Guide and TI Drivers API Reference.
